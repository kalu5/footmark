             通过大量的数据训练后，使其具备理解人类语言、思考、推理并输出人类语言的能力。

## 大模型部署

1. 本地部署Ollama
  优点：数据安全、自主可控、长期成本低
  缺点：初始成本高、需长期维护、性能受限（硬件需求高）
2. 官方开放api（deepseek、kimi、qianwen）
  优点：前期成本低、无需部署和维护、随时访问
  缺点：隐私不能保障、长期成本高、可控性差
3. 云服务平台（阿里百炼、火山引擎）
  优点：前期成本低、无需部署和维护、选择度高
  缺点：安全及隐私不能保障、长期成本高
  
## 大模型交互

会话记忆：处理方案（会话历史滚雪球）
将每次的结果再下次交互时发送过去

```py
#第一次：
{
  message: [
    {role: 'system', content: 'ai '},
    {role: 'user', content: '12个苹果4个人怎么分'},
  ]
}
#响应：
message: [
    {role: 'assistant', content: '每人3个'},
  ]

#第二次：
{
  message: [
    {role: 'system', content: 'ai '},
    {role: 'user', content: '12个苹果4个人怎么分'},
    {role: 'assistant', content: '每人3个'},
    {role: 'user', content: '3个人呢'},
  ]
}
#响应：
message: [
    {role: 'assistant', content: '每人4个'},
  ]

```

## 提示词工程

- 提示词：引导大模型进行内容生成的命令（一句话/一个问题等）
- 提示词工程：通过有技巧的编写提示词，使大模型生成尽可能符合预期的内容，这一持续的过程就称为提示词工程

- 坏的提示词：
  给我讲个故事
  用python写一个游戏

- 写一个好的提示词的关键步骤：

  角色：给大模型设定角色和能力
  任务：
    明确核心请求和任务
    按步骤拆解复杂任务
  要求：
    指定风格与语气
    明确要求输出格式
    提供输入输出示例
